# Poblacion Danao Bohol Clinic Appointment System

Full-stack clinic system with Django + React, JWT auth, WebSocket real-time notifications, and Fernet-encrypted medical records.

## Quick Start

### Backend

```bash
cd backend

# Create virtual environment
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Generate .env with Fernet key and secret key
python -c "from cryptography.fernet import Fernet; print('FERNET_KEY=' + Fernet.generate_key().decode())" >> .env
python -c "import secrets; print('SECRET_KEY=' + secrets.token_urlsafe(50))" >> .env
echo "DEBUG=True" >> .env
echo "ADMIN_SPECIAL_CODE=CLINIC2024" >> .env

# Run migrations
python manage.py makemigrations accounts appointments records messaging notifications
python manage.py migrate

# Create admin user
python manage.py shell -c "
from accounts.models import CustomUser
u = CustomUser.objects.create_superuser('admin', 'admin@clinic.com', 'admin123')
u.role = 'admin'; u.first_name = 'System'; u.last_name = 'Admin'; u.save()
print('Admin created.')
"

# Start server (Daphne for WebSocket support)
daphne -b 0.0.0.0 -p 8000 clinic.asgi:application
```

Or use the setup script:
```bash
chmod +x setup.sh && ./setup.sh
daphne -b 0.0.0.0 -p 8000 clinic.asgi:application
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Open http://localhost:5173

## API Endpoints

### Auth
- `POST /api/auth/register/` — Register patient or doctor
- `POST /api/auth/login/` — Patient/doctor login
- `POST /api/auth/admin-login/` — Admin login (requires special_code)
- `POST /api/auth/token/refresh/` — Silent token refresh
- `POST /api/auth/logout/` — Blacklist refresh token
- `GET  /api/auth/me/` — Current user info

### Users
- `GET  /api/users/` — List users (role-filtered per requester)
- `GET  /api/users/<id>/` — User detail
- `POST /api/users/<id>/deactivate/` — Soft delete (admin)
- `POST /api/users/<id>/restore/` — Restore user (admin)

### Appointments
- `GET  /api/appointments/` — List (role-filtered)
- `POST /api/appointments/book/` — Book appointment (patient)
- `POST /api/appointments/<id>/cancel/` — Cancel (any role, flowchart notifications)
- `GET  /api/appointments/stats/overview/` — Admin stats

### Prescriptions & Records
- `GET  /api/prescriptions/` — List prescriptions
- `POST /api/prescriptions/create/` — Create (doctor)
- `GET  /api/records/` — List medical records
- `POST /api/records/create/` — Create encrypted record (doctor/admin)

### Messaging
- `GET  /api/messages/` — Inbox
- `POST /api/messages/send/` — Send message
- `GET  /api/messages/thread/<user_id>/` — Thread view
- `GET  /api/messages/conversations/` — Conversation list

### Notifications
- `GET  /api/notifications/` — List notifications
- `POST /api/notifications/mark-all-read/` — Mark all read

### WebSockets
- `ws://localhost:8000/ws/notifications/<user_id>/` — Real-time notifications
- `ws://localhost:8000/ws/messaging/<user_id>/` — Real-time messaging

## Notification Rules (from flowchart)

| Trigger | Notified |
|---------|----------|
| Admin cancels appointment | Patient + Doctor |
| Doctor cancels appointment | Patient only |
| Patient cancels appointment | Doctor + all Admins |
| New appointment booked | Doctor |
| New prescription added | Patient |
| New message received | Receiver |

## Tech Stack

- **Backend:** Django 5.0, Django REST Framework 3.15.2, simplejwt 5.3.1
- **WebSocket:** Django Channels 4.0.0, Daphne 4.0.0
- **Encryption:** cryptography 42.0.0 (Fernet)
- **Frontend:** React 18, Vite 5, Axios, React Router DOM 6
- **Database:** SQLite3
